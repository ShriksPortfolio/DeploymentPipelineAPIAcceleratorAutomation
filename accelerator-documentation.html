Governance Scorecard

Six governance pillars scored by the accelerator's built-in capabilities.

Audit Trail 9/10

Every deployment generates a structured log with operation ID, timestamps, item-level status, error details, and deployment notes. Supports DataFrame export for downstream analytics.

Access Control 9/10

Service principal authentication eliminates personal credential usage. Scoped permissions via Entra ID ensure least-privilege access across workspaces.

Environment Isolation 9/10

Strict Dev → UAT → Production stage separation enforced through pipeline stages. Workspace assignments are validated before every deployment.

Change Traceability 8/10

Each deployment carries a unique operation ID, source/target item mapping, execution timestamps, and deployment notes. Full chain of custody for every change.

Reproducibility 7/10

Configuration-driven notebook — all deployment parameters are defined in a single cell. Identical results on every run with no environmental drift.

Approval Gates 6/10

First-time deployments require a one-time manual cloud connection mapping in the portal. Subsequent deployments are fully automated. Fabric pipeline approval rules can layer additional controls.
